# AF Docs

Цей docs-проєкт описує поточну реалізацію `af_app` і `af_api`: мобільний Flutter застосунок, FastAPI backend, PostgreSQL модель, файлові вкладення, Telegram-бота, Firebase push notifications і планувальник нагадувань.

## Що тут є

- C1-C4 діаграми архітектури.
- Sequence diagrams для auth, задач, коментарів, payroll і push-навігації.
- ERD для SQLAlchemy моделей і Alembic schema.
- OpenAPI reference, згенерований з FastAPI `app.openapi()`.
- Інструкції локального та Docker запуску docs.
- Опис `.env` конфігурації для backend і frontend.

## Межі документації

Docs не змінюють поведінку `af_app` або `af_api`. Вони читають поточний backend-код для генерації OpenAPI schema і зберігають результат у `apps/docs/openapi/af-api.json`.

## Основні частини системи

- `af_app` - Flutter client з `Provider`, `Dio`, `SharedPreferences`, Firebase Messaging і екранами задач, профілю, меню та payroll.
- `af_api` - FastAPI backend з JWT auth, SQLAlchemy async sessions, Alembic migrations, upload storage, Telegram Bot API, Firebase Admin SDK і APScheduler.
- PostgreSQL - основне сховище користувачів, задач, повідомлень, вкладень і payroll-звітів.
- `uploads/` - файлове сховище для task/comment/payroll media.
